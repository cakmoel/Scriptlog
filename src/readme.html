Welcome to Scriptlog

A simple, secure home for your stories and thoughts.

Scriptlog is a free and open-source PHP blog software. It is designed to be simple, private, and secure without the complexity of big Content Management Systems.

Scriptlog is built to:

  • Run your personal blog without needing a heavy CMS.
  • Keep your site secure with modern security practices.
  • Load fast with minimal resources.

Requirements

Before installing, make sure your web hosting supports the following:

  • PHP: Version 7.4 or higher
  • Web Server: Apache or Nginx
  • Database: MySQL 5.7+ or MariaDB 10.3+
  • HTTPS is recommended.

Installation

Step 1: Download Scriptlog

Download the latest version of Scriptlog as a ZIP file from the official website or GitHub repository. Extract the ZIP file on your computer, then upload all the files and folders to your web server using FTP or your hosting control panel's file manager.

Step 2: Create a Database

Log in to your hosting control panel (like cPanel) and create a new MySQL database. Make sure to write down the database name, username, and password. You will need them in the next step.

Step 3: Run the Installer

Open your web browser and go to the address where you uploaded Scriptlog, followed by /install/. For example, if your site is at http://example.com, go to http://example.com/install/.

Follow the installation wizard:

  • Step 1: The installer checks if your server meets the requirements.
  • Step 2: Enter your database details (name, username, password) and create an admin account.
  • Step 3: Complete the setup. The installer saves your settings automatically.

Step 4: Delete the Install Folder

For security, delete the install/ folder from your server after installation is complete.

After Installation

Once installed, you can access your site at the following addresses:

  • Your Blog: http://your-domain.com/
  • Admin Panel: http://your-domain.com/admin/

Log in to the admin panel with the account you created during installation to start writing posts, adding pages, and managing your blog.

Directory Structure

After uploading Scriptlog to your server, you will see these main folders:

  • admin/ — The administration panel where you manage your blog.
  • lib/ — Core application files (you do not need to edit these).
  • api/ — RESTful API endpoints for external integrations.
  • public/ — Contains your theme, uploaded files, cache, and logs.
  • install/ — The installer wizard (delete this after installation).

How It Works

When someone visits your blog, Scriptlog checks the web address and loads the right page. If the page does not exist, it shows a friendly 404 message. Your blog's appearance is controlled by a theme, and you can change it anytime from the admin panel.

Security

Scriptlog includes several security features to keep your blog safe:

  • Login protection with password hashing.
  • Form security to block spam and fake submissions.
  • Content filtering to prevent malicious code.
  • Data encryption for sensitive information.
  • User roles that control what each person can do.

User Levels

Scriptlog has six user levels with different permissions:

  • Administrator: Full access to everything.
  • Manager: Can manage content, comments, and settings.
  • Editor: Can manage content and comments.
  • Author: Can write and manage their own posts.
  • Contributor: Can write posts but cannot publish them.
  • Subscriber: Can only view the dashboard.
